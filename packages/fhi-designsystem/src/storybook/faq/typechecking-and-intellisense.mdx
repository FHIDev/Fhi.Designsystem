## Typesjekking & Intellisense

FHI Designsystem kan levere intellisense for ***VS Code*** og ***Webstorm***. Slik kommer du i gang:

### VS Code

1. Sett følgende inn i `.vscode/settings.json`
```json 
{
    "html.customData": ["./node_modules/@folkehelseinstituttet/designsystem/custom-elements.json"],
}
```

2. Installér [Web Component Language Server](https://wc-toolkit.com/integrations/web-components-language-server/)-utvidelsen i VS Code

### Webstorm

Designsystemet inkluderer en ```web-types.json```fil. WebStorm skal kunne lese denne filen uten at du trenger å gjøre noen grep.

> FHI Designsystem leverer foreløpig ikke type-filer for typesjekking eller intellisense for andre IDE-er enn VS Code og Webstorm. Det er også begrensninger i intellisense for komplekse typer, men det kommer opp forklaringer av typen.
> 
> Konsekvensene av dette er at du ikke vil få noen forslag fra intellisense relatert til komponentene sine attributter utenom standard forslag fra IDE-en, hvis du ikke bruker VS Code eller Webstorm. En eventuell kompilator vil ikke typesjekke komponentene.

Bedre støtte for typesjekking og intellisense kommer i fremtiden og vi har åpene issues relatert til dette.
