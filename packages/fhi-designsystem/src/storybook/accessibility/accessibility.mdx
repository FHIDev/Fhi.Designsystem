# Universell utforming

Innhold på nettsider og apper som skal ut til et generelt publikum fra offentlig sektor i Norge skal følge WCAG-standeren for universell utforming på nett.
Fagsystemer som bare er tilgjengelig for ansatte, er derimot ikke omfattet av kravene til universell utforming. Men det betyr ikke at disse systemene ikke kan dra nytte av prinsipper om universell utfordring innom ikt-løsninger.

Du kan lese mer om universell utforming og WCAG-standeren på [uutilsynet.no](https://www.uutilsynet.no/wcag-standarden/wcag-standarden/86).

## Farger

WCAG-standeren sitter krav angående bruk av farge og fargekontrast. Farger skal ikke benyttes som eneste visuelle virkemiddel til å formidle informasjon ( [1.4.1 Bruk av farge (Nivå A)](https://www.uutilsynet.no/wcag-standarden/141-bruk-av-farge-niva/93) ).
Visuelle komponenter i designsystemet bruker fargetemaer som møter kontrastkravene, men det er opp til hvert enkelt produkt å ta i bruk farger og kontrast på riktigt måte. See [Design Tokens] for veiledning på hvordan ta i bruk semantiske fargetokens fra designsystemet. 

## ARIA

Komponenter i designsystemet støtter bruk av [ARIA-attributter](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Reference/Attributes).
ARIA brukes for å hjelpe brukere av assisterende teknologi med å navigere nettsider og apper, for eksempel ved å sette aria-label på en ikon-knapp. 
Som hovedregel brukes HTML-elementer og attributter når det er mulig, og ARIA kan vurderes hvis HTML ikke har semantikken som trengs. **Husk: ingen ARIA er bedre enn dårlig ARIA**. 

Les mer om ARIA [her](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Reference/Attributes/aria-label).

```
<fhi-button type="button" aria-label="edit" icon-only>
    <fhi-icon-pencil></fhi-icon-pencil>
</fhi-button>
```

## Tastaturnavigering

Komponenter støtter tastaturinteraksjoner, slik som enter/space for å trykke på en knapp.
